Chapter 01 — The Question

What does a production-grade data pipeline actually look like?

Most analytics projects start with a clean CSV. Real pipelines are messier — data arrives daily, breaks unexpectedly, and needs to be tested, transformed, and monitored automatically. This project simulates what a live e-commerce data stack looks like end to end — daily order generation, idempotent ingestion into Snowflake, multi-layer dbt transformations with data quality tests, automated anomaly detection, ML forecasting, and a live dashboard with a natural language AI assistant. Every piece runs automatically each day without any manual intervention, which is the point: pipelines should run themselves.

Chapter 02 — The Method

Five phases, one automated chain

The pipeline is built in five sequential phases, each triggered only if the previous one succeeds. Prefect handles the orchestration on a daily cron schedule, with retry logic built into each flow. The data architecture follows a medallion pattern — Raw, Staging, and Mart — with dbt managing transformations and 36 data quality tests enforcing integrity at every layer. Anomaly detection uses Isolation Forest against daily order count and revenue, and ML forecasting runs two Prophet models: one for 30-day revenue, and twenty separate models for per-product demand over 7 days.

Chapter 03 — The Impact

Everything a data team needs, running without them

The entire pipeline — from raw order generation to updated dashboard — completes in roughly 3 minutes each day. The AI assistant on Page 4 uses a two-call Groq pattern: the first call generates SQL from a natural language question using the full mart schema as context, that SQL runs against Snowflake, and the second call translates the results back into a plain English answer. The forecast accuracy currently sits at a MAPE of 52% — expected given only 40 days of synthetic data with no underlying trend, and improving daily as the training window grows.
