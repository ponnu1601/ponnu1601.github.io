Chapter 01 — The Question

What happens when healthcare data stays locked away?

Healthcare administrators make decisions about costs, staffing, and treatment protocols every day — often without direct access to the data that should inform those decisions. Getting answers typically means waiting on a data analyst to write SQL queries, pull reports, and format results. The goal here was to remove that dependency entirely: build an interactive dashboard where any stakeholder could filter by condition, insurance provider, admission type, or gender and instantly see how the numbers shift.

Chapter 02 — The Method

A full data pipeline from MySQL to interactive web app

Patient records were loaded into a MySQL database and queried through 40+ optimized SQL queries using SQLAlchemy with caching for sub-second response times. The Streamlit frontend wraps these queries in a 6-section tabbed dashboard — covering demographics, medical conditions, insurance analysis, admission types, medication patterns, and a dataset overview. Session state management keeps all filters in sync across sections, so a selection made on one page carries through the entire analysis. One important note: this project uses a synthetic Kaggle dataset built for educational purposes, so findings reflect the dataset's characteristics rather than real-world healthcare benchmarks.

Chapter 03 — The Impact

Standardized costs and a billing pattern worth investigating

The analysis surfaces three notable findings. First, billing costs vary by only 2.5% across all six medical conditions — unusually uniform, which could reflect the synthetic nature of the dataset or genuine cost standardization worth benchmarking against industry norms. Second, there is a consistent $137 higher billing for one gender across all conditions and admission types — a pattern that warrants investigation for potential treatment pathway differences or equity concerns. Third, cancer shows the lowest average billing despite typically being among the most expensive conditions to treat, flagging a potential data quality gap. The dashboard puts these patterns in front of decision-makers directly, without requiring a technical intermediary.
